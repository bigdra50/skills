# 業務プロジェクト向け fragments

業務システム / Web アプリの Defer trigger 成立時に使う。
いずれも「実情報が埋まる分だけ」生成する。空欄だらけのテンプレを置かない。

## ONBOARDING.md

合格基準: 新メンバーが初日に環境構築〜初コミットを完了できること。

```markdown
# Onboarding

## Day 1 ゴール

{{動作確認できる状態の定義。例: ローカルで app が起動し、テストが通り、PR を 1 つ出せる}}

## 前提アカウント・権限

{{必要な権限・申請先。VPN / シークレット取得の手順がある場合はここ}}

## 環境構築

​```bash
{{1 コマンド (docker-compose up / devcontainer / 専用 CLI) を正とし、散文は補足だけ}}
​```

うまくいかないとき: {{既知のハマりどころ。発生したら追記していく}}

## 最初のタスク

{{good first issue の場所 / レビューの流れ / 質問チャンネル}}
```

## upgrade-guide/upgrade-v{{X.Y}}.md (破壊的変更リリース毎に凍結)

```markdown
# v{{X.Y}} へのアップグレード

- 対象: v{{前版}} からの移行
- 破壊的変更: {{一覧}}

## 手順

1. {{バックアップ}}
2. {{migration コマンド}}
3. {{確認方法}}

## ロールバック

{{戻し方。戻せない場合はその旨を明記}}
```

過去版のガイドは編集しない (版に凍結する。Grafana 型)。
